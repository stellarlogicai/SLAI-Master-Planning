# SLAI Product Ecosystem Map

**Status:** Strategic planning  
**Active build priority:** ServicesOS remains priority one.  
**Purpose:** Classify each SLAI product, module, and strategic branch into the correct ecosystem layer.

---

## Ecosystem Stack

```text
SLAI Core
↓
Security Foundation
↓
SLAI OS
↓
GrowthAI
↓
Product OS Layer
↓
Vertical Modules
↓
Overlap Linking
```

---

## Core Layer

### SLAI Core

Purpose:

- shared IDs
- tenant/user identity concepts
- event contracts
- audit concepts
- data linking rules
- cross-product references

Customer-facing?

```text
No. Infrastructure only.
```

### Security Foundation

Purpose:

- tenant isolation
- permissions
- audit logs
- AI approval gates
- secure integration boundaries
- incident visibility

Customer-facing?

```text
No. Infrastructure first.
Security products can be customer-facing later.
```
