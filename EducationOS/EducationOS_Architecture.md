# EducationOS Architecture

EducationOS should be its own vertical product, not mixed into ServicesOS.

## Shared SLAI Systems
Authentication, tenant management, roles, billing, Stripe Connect, messaging, notifications, documents, e-signatures, file storage, reporting, audit logs, AI framework, training engine, marketplace infrastructure.

## Education-Specific Modules
Students, parents, instructors, sessions, learning plans, subjects, goals, progress, attendance, lesson plans, assignments, quizzes, certifications, marketplace content, parent portal, teacher prep.

## Design Principle
Shared foundation + domain-specific modules.
