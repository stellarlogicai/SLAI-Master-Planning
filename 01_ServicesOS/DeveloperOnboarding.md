# ServicesOS Developer Onboarding

## Part 1 - Founder Vision

### Why ServicesOS Exists
ServicesOS exists to give cleaning businesses a single platform for customers, estimates, contracts, scheduling, employees, training, and payments.

Core beliefs:
- Fair pricing over maximizing short-term profit.
- Stability > Features.
- AI should reduce busy work, not replace people.
- Employees deserve respect and a voice.
- Long-term trust matters more than fast growth.

### Leadership Principles
- Lead by doing.
- No one is above the rules.
- Promote from within when earned.
- Hire for character and train for skill.
- Listen first.
- Data informs decisions.

## Part 2 - Technical Overview

### Stack
- Web App: React
- Employee App: React Native
- Backend: Firebase + Cloud Functions
- Database: Firestore
- Payments: Stripe + Stripe Connect

### Core Workflow
Lead -> Customer -> Estimate -> Contract -> Payment -> Schedule -> Assign Employee -> Employee App -> Checklist -> Photos -> Complete Job -> Final Payment

### Critical Rules
Never break:
- Payments
- Scheduling
- Security
- Customer Data
- Employee Workflow
