# Studio Pilot and Business Model

## Value Proposition
SLAI provides behavioral competitive integrity that can run alongside existing anti-cheat systems.

## Pilot Offer
Run a silent parallel test against the studio's existing anti-cheat.

Outputs:

- Detection comparison
- False positive analysis
- Evidence report
- Latency/performance report
- Recommendations

## Pilot Scope
Define:

- Game build
- Telemetry access
- Match sample size
- Player sample size
- Test duration
- Privacy rules
- Data retention
- Studio review process

## Success Metrics
- Detection rate
- False positive rate
- Precision
- Recall
- Reviewer agreement
- Latency
- CPU/GPU overhead
- Evidence quality
- Time to investigation

## Business Models
- Annual license
- Usage-based per MAU/match/telemetry volume
- Tournament add-on
- Enterprise integrity package
- Custom model tuning

## Studio Concerns
Studios will ask:

- How many false positives?
- What data is collected?
- Can players appeal?
- How much latency?
- What overhead?
- Can it run with current anti-cheat?
- Is the evidence explainable?
- How is privacy handled?

## Trust Strategy
Offer silent mode, human review, configurable thresholds, privacy-aware telemetry, explainable evidence, and studio-controlled enforcement.

## Final Principle
Studios do not only need detection. They need trust.
