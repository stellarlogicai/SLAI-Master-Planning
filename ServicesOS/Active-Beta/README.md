Status: Active planning

## Purpose

This folder stores active ServicesOS beta planning decisions before they become coding tasks in `stellarlogicai/ServicesOS`.

The goal is to keep wife-beta planning focused, practical, and safe:

- ServicesOS remains the priority-one active build.
- Beta work should support real cleaning-business operations.
- Future platform features should be documented without distracting from beta-critical workflows.
- Coding agents should not turn design notes into implementation unless Jamie explicitly promotes them to a ServicesOS issue or Codex task.

## Current Wife-Beta Focus

The first beta owner workflow should be simple enough to use daily:

1. Dashboard
2. Create Estimate
3. Customers
4. Schedule
5. Payments
6. Settings

The beta should verify that these visible areas actually work as planned, not just that the sidebar looks cleaner.

## Current Market-Entry Focus

After wife beta blockers are fixed, ServicesOS should not launch broadly.

The next market-entry goal is:

```text

keep using it after the first month,
and start proving ServicesOS can fund its own growth.
```

Current first external wedge:

```text
Small owner-operated cleaning businesses doing recurring residential cleaning,
move-out cleaning, real-estate prep cleaning,
and short-term rental turnover-style work.
```

## Important Rules

Route, crew, GPS, AI, payroll, employee app, and job-costing design is important, but it should be implemented in controlled layers.

Do not overbuild advanced optimization, payroll processing, employee app scope, or autonomous AI before the basic owner workflow is stable.

The first coding task should remain wife-beta admin sidebar cleanup and visible workflow verification. More advanced Schedule, Crew, Route Helper, AI Day Planner, Employee App, and Payroll Assist work should become separate promoted tasks after the foundation is stable.

## Market-Entry Guardrail

Do not run paid ads, buy lead lists, build SMS automation, build AI at scale, or broaden into other verticals before wife beta produces real workflow feedback and one narrow Founder Access test is ready.

Current rule:

```text
Real workflow first.
Wife beta second.
Founder Access third.
Expansion later.
```
