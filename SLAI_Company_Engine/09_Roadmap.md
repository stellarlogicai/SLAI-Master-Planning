# 09 — SLAI Company Engine Roadmap

**Status:** Future planning only  
**Active build priority:** ServicesOS remains priority one.  
**Purpose:** Define a phased roadmap for SLAI OS + GrowthAI + ServicesOS operating data as an internal company engine.

---

## Roadmap Principle

```text
Build only after ServicesOS beta is stable.
Start internal-only.
Use manual workflows first.
Automate only after real usage proves value.
```

---

## Phase 0 — Planning Blueprint

**Status:** Current planning phase

Goals:

- Define Company Engine purpose
- Link SLAI OS and GrowthAI planning
- Clarify ServicesOS data flow
- Define AI/human approval rules
- Define MVP boundaries
- Preserve future ideas without starting a new build

Outputs:

- Planning docs
- Roadmap
- Data/event concepts
- Founder workflow map
- Growth workflow map

---

## Phase 1 — Internal Manual Command Center

**Build timing:** After ServicesOS beta stability and explicit promotion

Goal:

Help Jamie run SLAI from one internal dashboard.

Possible features:

- Priority stack
- Product status cards
- Manual blocker tracking
- Manual decision log
- Manual lead list
- Manual follow-up dates
- Planning doc references
- Basic founder next-action list

No heavy automation yet.

---

## Phase 2 — GrowthAI-Assisted Lead Workflow

Goal:

Help SLAI find and follow up with ServicesOS prospects.

Possible features:

- Lead records
- Research notes
- AI-assisted lead summaries
- Outreach draft generation
- Human approval workflow
- Follow-up reminders
- Lead status pipeline

Guardrail:

```text
No autonomous outreach.
Jamie approves messages.
```

---

## Phase 3 — ServicesOS Data Connection

Goal:

Use real ServicesOS operating data to power company intelligence.

Possible connections:

- quote requests
- admin review status
- customer/beta feedback
- job status
- payment status
- lost leads
- recurring customers
- service performance

SLAI OS can surface operational blockers.
GrowthAI can surface follow-up and revenue opportunities.

---

## Phase 4 — Internal AI Agents

Goal:

Add limited AI assistance around already-proven workflows.

Possible agents:

- Founder Assistant
- Product Status Agent
- ServicesOS Beta Agent
- Research Agent
- Outreach Drafting Agent
- Company Memory Agent

Guardrail:

```text
AI recommends.
Jamie decides.
```

---

## Phase 5 — Customer-Facing GrowthAI Add-On

Goal:

Turn internal GrowthAI learnings into a future add-on for ServicesOS customers.

Possible customer-facing features:

- follow-up reminders
- lost lead recovery
- referral suggestions
- review request suggestions
- seasonal campaign ideas
- customer value insights
- recurring-service prompts

This should only happen after ServicesOS has enough real customer usage to validate the need.

---

## Phase 6 — Cross-Product Company Intelligence

Goal:

Connect more SLAI products through SLAI Core.

Possible sources:

- ServicesOS
- GrowthAI
- EducationOS
- RetailOS/POS
- ComplianceAI
- SLAI website
- Future modules

SLAI OS becomes the internal command layer across the full ecosystem.

---

## Long-Term Vision

```text
SLAI products create useful data.
GrowthAI turns data into growth opportunities.
SLAI OS turns signals into founder/team actions.
SLAI Core links everything cleanly.
```
