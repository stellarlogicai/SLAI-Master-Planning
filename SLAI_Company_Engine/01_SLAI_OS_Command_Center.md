# 01 — SLAI OS Command Center

**Status:** Future planning only  
**Active build priority:** ServicesOS remains priority one.  
**Purpose:** Define SLAI OS as the internal command center for Jamie and future SLAI operations.

---

## Core Role

SLAI OS is the internal command layer for the company.

It should not replace ServicesOS, GrowthAI, EducationOS, RetailOS, ComplianceAI, or FutureAI. It should help Jamie see across them.

```text
SLAI OS = company command center
```

It should answer:

- What needs attention today?
- What is blocked?
- What has changed?
- What is the active priority?
- What needs a human decision?
- What should be documented?
- What should be handed to Codex or a developer later?

---

## Primary Users

### Phase 1

- Jamie only

### Phase 2

- Jamie
- Future developer
- Future support person
- Future sales/growth helper

### Phase 3

- Internal SLAI team
- Possibly trusted contractors or role-limited users

---

## Core Dashboard Areas

### 1. Priority Panel

Shows the current company priority stack.

Example:

```text
1. ServicesOS beta hardening
2. Wife beta testing
3. UI cleanup
4. Payments stability
5. SLAI website
6. GrowthAI planning
```

This prevents future ideas from becoming accidental active tasks.

### 2. Product Status Panel

Shows current status by product:

- ServicesOS
- GrowthAI
- SLAI website
- EducationOS
- RetailOS / PharmacyOS
- ComplianceAI
- FutureAI

Each product should have:

- status
- current phase
- next action
- blocker
- owner
- last updated date

### 3. ServicesOS Beta Panel

Because ServicesOS is priority one, SLAI OS should eventually surface:

- beta blockers
- wife feedback
- customer workflow status
- admin workflow status
- Stripe/payment status
- UI issues
- validation/commit references

### 4. Founder Workflow Panel

Shows Jamie what to do next:

- today’s focus
- high-priority decision
- pending documentation
- open questions
- GitHub/Codex handoff needs
- follow-up tasks

### 5. Growth Opportunities Panel

Powered by GrowthAI later:

- leads to research
- follow-ups due
- referral opportunities
- cold leads
- customer opportunities

### 6. Company Memory Panel

Shows durable context:

- founder principles
- product rules
- active priorities
- decisions made
- roadmap notes
- planning repo references

### 7. Alerts and Incidents Panel

Shows anything needing attention:

- beta issue
- customer complaint
- failed automation
- overdue follow-up
- blocked development task
- urgent payment/support issue

---

## What SLAI OS Should Not Become Early

Avoid turning SLAI OS into:

- a full project management clone
- a full chat platform
- a full CRM
- a full analytics BI tool
- a replacement for GitHub
- a replacement for ServicesOS

The first value is focus and decision support.

---

## MVP Shape Later

The eventual MVP should be simple:

```text
Founder dashboard
Product status cards
Priority stack
Manual notes/decisions
ServicesOS beta status
GrowthAI lead/follow-up summary
Planning repo references
Next-action recommendations
```

No heavy automation at first.

---

## AI Role

AI can help by:

- summarizing status
- suggesting next actions
- detecting stale tasks
- summarizing feedback
- drafting Codex prompts
- connecting related planning docs

AI should not make final business decisions without Jamie.

```text
AI recommends.
Jamie decides.
```

---

## Relationship to Existing Products

SLAI OS is not a product vertical. It is the internal operating layer.

```text
ServicesOS creates operating data.
GrowthAI creates revenue opportunities.
SLAI OS surfaces what matters.
```

---

Do not build until ServicesOS beta is stable and Jamie explicitly promotes SLAI OS to active work.
