# 05 — Founder Workflow

**Status:** Future planning only  
**Active build priority:** ServicesOS remains priority one.  
**Purpose:** Define how Jamie would use the SLAI Company Engine day to day.

---

## Core Goal

The founder workflow should reduce mental load and help Jamie stay focused.

It should answer:

```text
What matters today?
What is blocked?
What needs a decision?
What can wait?
What should be documented?
What should be handed to Codex or a future developer?
```

---

## Daily Founder Flow

```text
Open SLAI OS
↓
Review priority stack
↓
Check ServicesOS beta status
↓
Review blockers and pending decisions
↓
Review GrowthAI opportunities
↓
Choose today's main action
↓
Document decisions
↓
Move only approved tasks forward
```

---

## Priority Stack

SLAI OS should always show the current priority stack.

Example:

```text
1. ServicesOS beta hardening
2. Wife beta testing
3. UI cleanup
4. Payments stability
5. SLAI website
6. GrowthAI planning
7. EducationOS planning
```

This prevents future ideas from taking control of the workday.

---

## Decision Capture

Jamie makes many important product decisions while brainstorming.

SLAI OS should eventually capture:

- Decision made
- Reason
- Product affected
- Priority impact
- Active or future planning
- Related docs
- Next action

Example:

```text
Decision:
POS should be an add-on before RetailOS linkage.

Reason:
ServicesOS already supports payments and operational inventory.

Impact:
Future modules can use ServicesOS + POS first.

Status:
Future planning only.
```

---

## Codex / Developer Handoff

SLAI OS should help convert planning into safe coding tasks later.

A handoff should include:

- Scope
- Files to touch
- Files to avoid
- Acceptance criteria
- Validation commands
- What Codex/developer should report back

This matches the ServicesOS build style:

```text
Small task.
Clear acceptance criteria.
Validation required.
No green checks = no commit.
```

---

## Growth Workflow

GrowthAI should feed Jamie opportunities, not distractions.

Example:

```text
GrowthAI finds 5 possible cleaning businesses.
Research Agent summarizes 2 best fits.
SLAI OS shows 1 recommended outreach action for today.
Jamie approves/edit/sends.
```

The system should not overwhelm Jamie with 100 leads at once.

---

## Planning Workflow

When Jamie has a future idea:

```text
Capture idea
↓
Classify product area
↓
Mark active vs future
↓
Document if useful
↓
Keep ServicesOS priority clear
```

Future ideas are allowed. They just need to be parked properly.

---

## What This Should Prevent

- Losing important ideas
- Forgetting decisions
- Repeating the same planning work
- Spending Codex credits on unclear prompts
- Starting too many active projects
- Mixing future planning with current ServicesOS work
